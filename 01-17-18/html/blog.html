Also, I’ve written a piece of Javascript to automate the margin-top for this page. I feel really proud of it even though it’s a rather simple function. This is the first piece of Javascript that I have written that I have gotten to implement in a real world application instead of just practicing on Codewars, or from a YouTube tutorial. Check out the code below;

 

function height() {

var blogContainer = document.getElementById("blog-container");

var blogContainerHeight = document.getElementById("blog-container").clientHeight;

var marginHeight = blogContainerHeight - 400;

blogContainer.style.marginTop = marginHeight + "px";

}

height();

"Playing with JS" by Straight to Web

12/22/2017

Playing with JS Playlist

I’ve been trying to learn Javascript lately. I have been doing the lessons on the free version of Codecademy, but they lack certain real world applications, and ways to utilize JS with webpages. So I found this tutorial from Straight to Web on YouTube. It’s a quick little tutorial showing you how to build a webpage that changes background color when the user interacts with certain parts of the page. I followed it’s instructions closely and found a couple of useful little tidbits. I also learned a couple overarching JS lessons by using it. Here I will explore what I learned and how it is helping me with JS in general. Also I have made some changes to his original lesson. I felt the UI/UX could be a little cleaner, and some of the JS code was a little redundant, hence the commented out section that you will see below.

Alright, so what did I learn from this exercise? First off, the Developer Tools in Chrome are indispensable when troubleshooting the code. I ran into several instances where I had mistyped some small word. Like, typing getElementByID instead of getElementById, or document.body.sytle instead of document.body.style. These threw me for a loop and had me searching my code at the lines specified in the Chrome dev tools to find the particular error that it was referencing.

The other error I received was “Uncaught TypeError: Cannot read property 'addEventListener' of null”. After googling the error I discovered that I needed to move the <script></script> tag to a location after the Ids/Classes/Etc that the script is looking for, or else it will not find the necessary elements. I originally had the <script></script> tag in the header, when I moved it to below the last closing </div> tag the problem was resolved.

I commented out the changeBackgroundColor function that he adds in the beginning of the lesson for two reasons. I felt that it was unnecessary to have two functions for changing the background color, and once I had moved the color input to inside the main <div> of the page it created a conflict. Clicking the color input caused the prompt to appear as well as the color selector. So, I removed it and I solved both problems with a single action.

The last useful tidbit that I learned is the usage of JSLint.com. It helped me pick out small errors with in my code and to ensure the use of proper syntax.
